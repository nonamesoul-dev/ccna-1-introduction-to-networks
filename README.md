### What you can DO in practice

- Understand why a network works or does not work (not just memorizing the OSI model).
- Identify which layer is failing (physical, network, transport, service).
- Detect common issues such as:
  - Incorrect IP address
  - Misconfigured default gateway
  - DNS resolution failures
  - Physical connectivity or cabling problems
- Use basic diagnostic commands:
  - `ping`
  - `ipconfig` / `ifconfig`
  - `tracert` / `traceroute`
- Read and understand simple network topologies without losing context.
